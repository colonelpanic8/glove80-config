## What this is
Glove80 keyboard running a custom RMK (Rust) firmware stack, replacing ZMK. Components:

- `rmk/glove80/` — firmware wrapper crate (central=LH, peripheral=RH).
  `nix develop --command ./build.sh` → two UF2s. RMK is pinned as the
  `dependencies/rmk` submodule at `75d0edc3` on fork branch `glove80-rynk`.
- `tools/glove80-control/` — hybrid host CLI: Rynk owns keymap operations;
  product protocol v1.4 owns version, bootloader, lighting, and lighting config.
  89 tests.
- Lightbench — browser IDE for lighting/keymap. Lighting uses the product
  protocol; keymaps use the generated Rynk WASM client.
- Conditional lighting: gates + firmware-state conditions (VBUS etc.), compositor, records.
- Keyboard config: unified TOML pushed via CLI; keymap batching for hardware limits.

## Gotchas
- Rust and libclang are supplied together by the root flake at Rust 1.97.0;
  run builds/tests through `nix develop --command ...`.
- RMK-config prints DFU auto-layout warnings because nRF52840's chip defaults
  include a `[dfu]` table even when the `dfu_nrf` Cargo feature is disabled.
  This build uses `shared_flash` (mutually exclusive with `dfu_nrf`), so the
  explicit `0xec000`/8-sector RMK storage settings are still the generated
  layout; the warnings are feature-unaware diagnostics.
- Rynk dangerous operations use a physical unlock chord at `(0,0)` + `(0,13)`
  (outer top-row F1/F10 positions); normal keymap/config access stays open.
- Lightbench currently needs two WebHID sessions for the whole product:
  product-protocol HID for lighting and Rynk HID for keymaps.
- Bootloader flash: peripheral first, then central
  (`--peripheral --yes`, wait for GLV80RHBOOT/GLV80LHBOOT mounts).
- Keep untracked files out of monorepo root or builds stamp `-dirty`.
